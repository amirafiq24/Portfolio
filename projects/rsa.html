Secure File Transfer

This project implements a secure client–server file transfer system designed to enforce confidentiality, integrity, and authentication over an untrusted network. The system establishes a secure session using RSA public key cryptography to exchange a symmetric session key, which is then used to protect all communication. User authentication is handled using the Secure Remote Password protocol, ensuring passwords are never transmitted or stored in plaintext. Files are encrypted end to end using authenticated encryption with keys derived via PBKDF2. The project emphasizes practical cryptographic protocol design, secure key management, and real-world attack resistance.

Overview

  • Designed a secure client server communication protocol
  • Ensured data and passwords are protected during communication
  • Designed the system to work safely over untrusted networks

Implementation

  • Used RSA to securely establish shared session keys
  • Applied SRP to authenticate users without sending passwords
  • Encrypted all file transfers to protect data in transit

Results

  • Successfully transferred files securely between clients
  • Authenticated users without exposing sensitive credentials
  • Achieved secure communication over untrusted channel

What I Learned

  • Applying cryptography concepts to real world systems
  • Designing secure authentication without storing plaintext passwords
  • Balancing security requirements with system simplicity

Built With:

Python, RSA, SRP, AES-CCM, PBKDF2, Sockets, Cryptography
